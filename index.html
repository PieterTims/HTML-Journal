Chapter 2 - Learning CSS

CSS cheatsheet CSS Properties Refererence

CSS syntax looks like this = selector{property:value;} , it is possible to add multiple property:value (or rule) into one selector.

  • selector is the "Who?" as in which tag is it pointing towards
  • property is the "What?" as in what property needs to be changed
  • value is the "How?" as in how do you want to change the property.

There is always a default css style that is applied to all tags, and when we write a css code corresponding to that default, it will override the default and use our code. The best practice to keep the multiple rules organised is to keep it in alphabetical order, which will make it easier to both read and debug.

There are other kinds of selectors as well, namely class and id selectors. These can be used to differentiate similar tags so that it is possible to have the same tag used for different things having different set of rules (Example: <img class="one"> and <img class="two"> can be styled by different selectors).

class & id selectors have higher priority.

ids can be used only once in each file, i.e. only one id called apple or only one id called carboard_box etc. , making it a super specific instance. One tag can have multiple classes embedded but only one id. CSS can be done in three different ways: Inline, Internal or External.

Inline CSS applies style inside a tag by adding a style component inside the opening tag. Examples:

  1. This one adds a bg color of blue violet and a margin of 10 pixels on the list while a <span> tag applies the color white to the text.
  2. While this one changes the list-style-type to georgian with a bg color of violet, with margin-right set to 100px.

Internal CSS applies style by using a <style> tag that resides in the <head> component. Each chapter resides in a <div> tag, and using the <style> tag, I added a different into chapter 2. Internal CSS applies to the whole page/html file.

External CSS on the other hand can be implemented into multiple html files. The way external CSS works, is the same as Internal CSS. You create a CSS file the same way you make a html file. Then, you use <link> to link that CSS file to a HTML file. External CSS is usually recommended to implement all style to your whole website.

*Priorities for CSS = Inline > Internal > External.

Chapter 3 - Everything Else HTML and CSS

  • Use the chrome dev tools to debug and change the the css part of the code easier. Right-clicking on a certain element and choosing inspect brings up the code for that exact element.
  • Favicons can be linked using rel="icon", there usually more than one resolution that needs to be saved for these favicons. Android, Chrome, Opera and Apple all have their own size. favicons are used to make it easier for users to identify which website they are in just from a glance.
  • <div> is used to defined a division or section of an HTML. This tag is a block-level organizing tool.
  • <span> is used as an inline container to style a part of a text or document. This tag is an inline-level organizing tool.
  • each element of an HTML has a default CSS style that is implemented by chrome. block-level elements have multiple display elements that can be manipulated. content sits inside a padding which sits inside a border, and there is a margin that sit outside of the border. for easier context:

    This is its content.

    Here, the text inside has been set to 10pt, with padding 10pt, border style: double with a width of 5 pt, and a margin of 1 inch.
  • Display is one of the default CSS styles written to the tags, this specifies the display behavior of an element. There are 4 commonly used display properties. Block, Inline, Inline-block, and None.
    1. Block is usually seen in <p> and <h>. This displays an element as a block element.It starts on a new line, and takes up the whole width
    2. Inline is usually seen in <span>,<a> and <img>. This displays an element as an inline element. Any height and width properties will have no effect
    3. Inline-block displays an element as an inline-level container, but you can add height and width properties to it.
    4. None just removes the element from displaying. There is a property called visibility that can be added to an element to hide it while still taking the space of the display it implements.
